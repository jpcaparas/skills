# Gotchas

1. **Do not turn a spreadsheet request into prose.** This skill is for workbook generation. If the user only wants a written itinerary, use another tool.
2. **Do not ask endless questions.** Read the files first, then ask one blocker batch or none.
3. **Do not silently fix contradictions.** Show them as review flags and keep the workbook honest.
4. **Do not bury shopping.** If the user cares about shopping, it needs to stay visible in Daily Plan, the options bank, and Buy List.
5. **Do not duplicate Pack and Buy.** The distinction is operationally useful; keep it clean.
6. **Do not create geography nonsense.** Avoid cross-city zig-zagging, three distant clusters in one block, or return-day heroics.
7. **Do not overload Prep & Compliance with generic travel checklists.** Keep only the tasks that matter for this trip.
8. **Do not make Tokyo-specific assumptions for every trip.** Preserve the workbook architecture, but adapt destination-specific text and status labels sensibly.
