# Typography And Copy

Type does more than style the interface. It sets pace, trust, and cognitive load.

## Role Stack

Define roles before fonts:

1. **Display**
   - used sparingly for major statements or hero moments
2. **UI Sans**
   - labels, controls, navigation, tables, forms
3. **Body**
   - paragraphs, help text, descriptions
4. **Mono**
   - code, timestamps, IDs, metrics, aligned numbers

One family can cover multiple roles, but the roles must still feel distinct.

## Choosing Type By Product

### Editorial or narrative products

- allow more character in the display face
- serif accents are acceptable when they support reading or tone
- keep UI text quieter than headlines

### Product apps and dashboards

- default to high-quality sans for display and body
- use mono only where alignment earns it
- avoid decorative serif moments in dense operational views

### Desktop tools

- optimize for long sessions and repeated scanning
- favor clarity at smaller sizes
- keep labels decisive and avoid tiny low-contrast metadata

## Scale Rules

- Headline scale should reflect importance, not ego.
- Reading measure should stay comfortable.
- Helper and error text must remain readable, not merely smaller.

Useful defaults:

- keep body copy around a comfortable reading width
- keep interface labels short and specific
- keep table type dense but not starved

## Labeling Rules

Good labels:

- name the thing
- distinguish similar controls
- avoid filler verbs

Weak labels:

- "Optimize"
- "Manage"
- "View Details"
- "Learn More"

Prefer:

- "Retry sync"
- "Export CSV"
- "Pin column"
- "Show failed jobs"

## Copy Tone

Strong interface copy is:

- specific
- calm
- believable
- short enough to scan

Avoid:

- startup cliches
- vague premium language
- invented certainty
- generic placeholder names and perfect round numbers

## Number And Data Handling

Use mono when it improves comparison:

- timestamps
- IDs
- currency columns
- percentages in dense lists

Do not use mono for:

- long descriptions
- explanatory empty states
- narrative onboarding copy

## Common Fixes

If the UI feels amateur:

- tighten the difference between primary and secondary text roles
- reduce the number of text sizes in play
- shorten labels before shrinking fonts
- let supporting copy breathe under strong headings
- remove decorative all-caps metadata unless it adds actual structure
