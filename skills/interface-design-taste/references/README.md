# Reference Index

This skill is organized as a hub and spoke. Load the smallest file that answers the task.

## Reading Guide

| Task | Read |
| --- | --- |
| Define the design direction | `principles.md` |
| Choose a visual family | `style-families.md` |
| Fix hierarchy and composition | `layout-and-rhythm.md` |
| Fix typography and tone | `typography-and-copy.md` |
| Fix palette, surfaces, imagery, or icons | `color-material-and-iconography.md` |
| Fix motion and interaction states | `interaction-motion-and-states.md` |
| Adapt a design to web or desktop | `platform-adaptation.md` |
| Audit an existing interface | `critique-workflow.md` |
| Capture tokens and component rules | `design-systems-and-tokens.md` |
| Diagnose why the work still feels wrong | `gotchas.md` |

## Load Order

1. Start with `principles.md` for any new direction.
2. Add exactly one or two specialized references based on the current failure mode.
3. Use `critique-workflow.md` when working on an existing product instead of jumping straight to style changes.
4. Use `design-systems-and-tokens.md` only after the screen logic is clear.
