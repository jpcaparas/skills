# Gotchas

These are the failure modes that make polished-looking work still feel wrong.

1. **The hero is better than the product.**
   - Symptom: the landing page looks premium, but every inside screen collapses into generic cards.
   - Fix: design the working surface first, then let marketing inherit from it.

2. **The interface is composed of containers, not priorities.**
   - Symptom: everything is boxed, yet nothing feels organized.
   - Fix: re-establish dominant and supporting zones before choosing card styles.

3. **Accent color is being used as a substitute for hierarchy.**
   - Symptom: buttons, tags, links, charts, and illustrations all compete with the same color.
   - Fix: reduce accent usage to actions, focus, and the single most important highlights.

4. **Desktop surfaces are being treated like phone mockups.**
   - Symptom: oversized cards, giant spacing, hidden secondary tools.
   - Fix: add density, pane logic, and precision affordances.

5. **The work is visually soft but functionally hard.**
   - Symptom: warm shadows and rounded cards hide weak navigation and poor labeling.
   - Fix: sharpen structure before softening the material language.

6. **Every section is trying to be the designed moment.**
   - Symptom: each region uses a different pattern, radius, or motion trick.
   - Fix: let most of the interface stay disciplined so a few areas can carry character.

7. **Motion exists, but state logic does not.**
   - Symptom: nice reveals, weak feedback.
   - Fix: finish hover, focus, loading, success, and error before adding choreography.

8. **The interface looks premium only with fake content.**
   - Symptom: real labels wrap badly, tables break, and actual data ruins the rhythm.
   - Fix: test with long labels, messy numbers, and realistic empty states.

9. **Typography is trying to do brand work and UI work with the same voice.**
   - Symptom: the same dramatic display treatment appears in forms, settings, and data tools.
   - Fix: separate display personality from UI clarity.

10. **Shadows are doing the work of grouping.**
    - Symptom: subtle depth everywhere, but weak relationships between objects.
    - Fix: use alignment, spacing, and headings as the primary grouping tools.

11. **Dark mode is just an inverted light mode.**
    - Symptom: contrast feels harsh and all surfaces flatten together.
    - Fix: retune neutrals, text contrast, border strength, and emphasis rules separately.

12. **Charts are louder than the decisions around them.**
    - Symptom: every series is saturated and the surrounding screen has no visual hierarchy.
    - Fix: quiet the chart palette and let one message carry the emphasis.

13. **The system is locked too early.**
    - Symptom: tokens are clean, but the product still feels generic.
    - Fix: return to `principles.md` and reset the direction before expanding the system.

14. **The critique is styling-only.**
    - Symptom: feedback starts and ends with colors, fonts, and shadows.
    - Fix: run the full `critique-workflow.md` order and diagnose the screen logic first.

15. **The design feels borrowed, not inevitable.**
    - Symptom: it resembles a trend board more than the product itself.
    - Fix: restate the product job and cut any motif that does not support it.
